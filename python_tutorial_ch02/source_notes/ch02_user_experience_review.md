# 第2章真实用户体验审阅与优化记录

审阅日期：2026-06-18  
审阅对象：`chapters/ch02_python_data_types.md` 及配套图片、脚本、README、manifest

## 总体结论

原稿的优点很明确：比喻丰富，图片数量充足，能把 Python 数据类型和学习记录、Stroop 实验数据、终端运行证据连接起来，不是单纯罗列语法。对初学者来说，这种“材料 - 容器 - 小项目”的主线是有效的。

主要问题也很集中：部分成果图出现得太早，读者还没学到列表、字典、小项目和运行证据，就已经看到 PowerShell 总览图，理解负担被提前抬高；个别段落还保留“课件里”“MATLAB 教材”等来源视角，会让教程读者从学习状态跳出来；部分小知识点对新手缺少“为什么会错”的提醒，例如字符串数字与整数的区别、`replace()` 的返回值、`round(2.5)` 的特殊舍入、嵌套列表示例里数字被写成字符串等。

本次修改后的版本将图文顺序调整为“先概念、再操作、再项目、最后证据”，并补充了新手更容易踩坑的解释和练习参考代码。

## 原稿中影响阅读体验的具体问题

| 维度 | 原稿问题 | 对新手的影响 | 本次处理 |
| --- | --- | --- | --- |
| 图片与正文顺序 | PowerShell 运行图和运行证据图原本放在变量章节后面 | 读者还没学小项目和脚本产物，就要理解 `dict`、`list`、报告文件、ready 检查 | 移到本章小项目后；运行证据图移到全部项目脚本介绍完成后 |
| 图文对应 | 布尔逻辑开关图放在 `and/or/not` 正文之前较远的位置 | 图里讲的是逻辑组合，但正文还在讲单个布尔值 | 移到 `and、or、not` 小节表格后 |
| 列表到字典过渡 | IBM 打孔卡分拣机在列表小节中提前解释 list 与 dict | 字典尚未正式出现，容易提前分心 | 调整为列表操作之后、字典章节之前的过渡图 |
| 来源口吻 | 出现“课件里”“参考 MATLAB 教材”等表达 | 读者会感觉自己在读改写说明，而不是正式教程 | 改成面向学习者的直接叙述 |
| 类型深度 | 数据类型地图有概念，但缺少 `type()` 的即时验证 | 新手容易把 `"86"` 和 `86` 混为一谈 | 新增 `type()` 检查示例，强调展示型字符串和可计算数字的区别 |
| 变量命名 | 实验时间变量没有单位 | 容易混淆毫秒、秒、次数 | 改为 `fixation_duration_ms` 和 `total_time_ms` |
| 数值取整 | 只提示银行家舍入，没有可运行例子 | 新手不知道该怎么验证 | 补充 `round(2.5)` 与 `round(3.5)` 的具体结果 |
| 字符串替换 | `replace()` 的返回值特性没有说明 | 新手可能以为原字符串会自动改变 | 补充“返回新字符串，建议重新赋值”的说明 |
| 嵌套列表 | 示例用 `"5"`、`"2"`、`"1"`，但讲的是列表结构 | 学生可能误以为数字都应该加引号 | 改为 `[5, 2, 1]`，并改用更清楚的变量名 `number_list`、`word_list`、`nested_list` |
| 练习反馈 | 练习 2-4 只有题目，没有参考实现 | 自学者做完后难以自查 | 新增参考代码，保留开放性练习给类型罗盘和 Stroop 数据包 |

## 修改后内容深度评估

修改后的深度更适合作为“第2章详细版”：它没有把内容扩到超出初学范围的类、迭代器、可变对象深水区，但补足了入门阶段最关键的判断力：如何区分类型、如何选容器、如何把类型组合进小项目、如何从报错反查类型问题。

仍建议后续章节继续补充两条线：第一，等学生学到函数时，再正式解释“可变对象”和“参数传递”；第二，等学生学到文件和表格时，再把本章 Stroop 数据包接到 CSV/JSON 读写和简单统计上。这样不会在第2章过早加重负担，又能保持跨章节连续性。
