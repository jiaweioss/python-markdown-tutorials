## 设计标准

- 文字面向初学者，先讲故事和类比，再落到概念、代码和可运行产物。
- 图片使用本地正式图，避免远程图片失效、显示不稳定或排版歪斜。
- 互联网配图必须下载到本地，并记录来源与授权。
- 每章保留 `source_notes` 来源记录和质量验收记录。
- 每章提供 `scripts/check_links.py` 检查本地图片引用。
- 正式图片优先使用统一宽度、统一留白和简洁构图；图片居中显示，长解释放在 Markdown 正文和图注中。
- 关键概念尽量配套 Python 生成的图表或真实运行截图，既服务理解，也保证投屏和分享时的审美稳定。
