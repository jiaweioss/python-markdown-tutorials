# 第2章类型选择卡片

看到一份新数据时，先不要急着写代码。先问：它是什么、会不会变、要不要计算、要不要查找。

## bool

- 判断问题：这是不是一个判断？
- 使用场景：保存 True / False，例如是否通过、是否完成、是否需要复习。
- 代码例子：`passed = average_score >= 60`

## int / float

- 判断问题：这是不是数量、得分、时长或比例？
- 使用场景：保存可以计算的数字。整数适合次数和编号，小数适合平均分和反应时。
- 代码例子：`time_ms = 523.7`

## str

- 判断问题：这是不是一段文字？
- 使用场景：保存姓名、题目、路径、标签、备注等文本信息。
- 代码例子：`student_name = '小明'`

## list

- 判断问题：这是不是一串有顺序的数据？
- 使用场景：保存多次测验分数、实验 trial、单词清单等可以逐个遍历的内容。
- 代码例子：`scores = [86, 92, 78]`

## dict

- 判断问题：这是不是查找表？
- 使用场景：保存一名学生、一张卡片、一次实验或一个文件的完整记录。
- 代码例子：`student = {'name': '小明'}`

## None

- 判断问题：这里是不是暂时没有值？
- 使用场景：保存“还没有结果”的状态，不要用空字符串或 0 假装缺失。
- 代码例子：`review_date = None`
